# ☕ CUP咖啡 · 知识库搭建实践

> 教学项目：从11份原始素材到可问答知识库的完整链路

---

## 项目概述

这是一个**咖啡店知识库搭建**的教学练手项目。提供了11份真实的门店运营素材（**充满典型陷阱**），演示如何通过系统性的数据清洗，将它们转化为分层级、可检索、安全可控的知识库。

## 项目结构

```
├─ 原始素材/              ← 11份原始文件（含5类典型陷阱）
│
├─ CUP咖啡知识库/          ← 所有清洗产出
│  ├─ 清洗决策清单.md      ← 逐份素材的处理记录
│  ├─ 问答测试平台.html    ← 交互式验证工具（浏览器打开即可使用）
│  ├─ slides_optimized.json ← PPT源文件（可重新生成）
│  ├─ 优化版PPT.pptx       ← 20页教学演示文稿
│  │
│  ├─ 公开层/  6条        ← 顾客可见，AI可直接回答
│  ├─ 内部层/  7条        ← 仅员工/管理层可访问
│  └─ 机密层/  2条        ← 仅店长+财务可访问
│
└─ .gitignore
```

## 素材中的5大陷阱

| 风险类型 | 涉及素材 | 问题描述 |
|----------|----------|----------|
| 🔴 版本冲突 | 员工手册2024版 vs 2026版 | 休1天 vs 休2天，内容自相矛盾 |
| 🟡 内容重复 | 客诉处理简略版 vs SOP完整版 | 3步大纲覆盖完整7步流程 |
| 🔴 时效混淆 | 2025年vs 2026年促销方案 | 过期方案误导顾客 |
| 🔴 机密泄露 | 咖啡豆采购价格表 | 采购底价直接暴露 |
| 🟡 测试误用 | 桂花拿铁糖浆用量记录 | 测试数据15ml被当作正式10ml引用 |

## 清洗流程

1. **素材盘点** — 逐份登记内容、时间、权限等级
2. **数据清洗** — 废弃重复、标注时效、隔离机密、标记测试数据
3. **权限分级** — 公开/内部/机密三层隔离
4. **结构化处理** — OCR转文本、长文本拆分、统一格式
5. **知识分类** — 按业务域分组
6. **质量校验** — 版本冲突检查、权限确认
7. **入库上线** — 每条知识带元数据标签

## 如何体验

1. 打开 `CUP咖啡知识库/问答测试平台.html`（浏览器运行）
2. 切换 **"未经清洗"** 模式 → 点击测试问题 → 看AI如何犯错
3. 切换 **"已清洗"** 模式 → 同样的问题 → AI回答正确安全
4. 切换 **"对比模式"** → 左右分栏亲眼见证差异

## 项目背景

本项目是为"知识库搭建"教学场景设计的实操案例。所有素材均来自模拟场景，保留了真实工作中常见的文档问题——残缺、重复、过期、机密混放、标注不全——让学习者亲身体验知识库建设从原始素材到成品交付的完整闭环。
